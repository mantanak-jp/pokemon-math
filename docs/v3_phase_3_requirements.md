# PokeMath V3 Phase 3 要求仕様

作成日: 2026-06-07  
対象: `v3/`  
位置づけ: V3 main昇格前のクイズ拡張要求仕様

## 1. 目的

V3 Phase 3では、PokeMathのクイズ機能を拡張する。

主目的は以下。

```text
1. 既存の算数クイズ仕様を維持する
2. 算数クイズに割り算を追加する
3. 算数以外のクイズとして国旗クイズを追加する
4. 数値専用だった4択クイズ構造を、文字列・画像を扱える汎用4択クイズ構造へ拡張する
5. 割り算・国旗クイズまで実装・実機確認できた段階で、V3の main 昇格判断へ進める状態にする
```

Phase 3では、時間クイズ、国・首都クイズ、都道府県クイズ、都道府県の形クイズ、矢印・迷路クイズは実装しない。  
これらは将来拡張案として残す。

## 2. V3 Phase 3 の実装対象

Phase 3で実装対象とするのは以下。

```text
実装対象:

1. クイズ種別選択
   - さんすうクイズ
   - こっきクイズ

2. さんすうクイズ
   - レベル1: 1けたの たし・ひき
   - レベル2: 2けた と 1けた
   - レベル3: 2けたの たし・ひき
   - レベル4: かけざん 九九
   - レベル5: 九九の わり算
   - レベル6: 2けた ÷ 1けた

3. こっきクイズ
   - 国旗を見て、正しい国名を4択で選ぶ
   - 難易度設定は設けない
   - クイズ種別選択後、そのまま問題開始する

4. Firestore国マスター
   - country_masters/{countryId}
   - REST Countries等から取得できる国・地域データを広く取り込む
   - 出題対象は enabled フラグで制御する
```

## 3. V3 Phase 3 では実装しないもの

以下は、V3 Phase 3の実装対象外とする。

```text
将来拡張案:

1. 時刻・時間クイズ
   - 何分後
   - 何分前
   - あと何分

2. 国・首都クイズ
   - 国名から首都を選ぶ
   - 首都から国名を選ぶ

3. 都道府県クイズ
   - 県庁所在地
   - 地方
   - 都道府県名

4. 都道府県の形クイズ
   - シルエット画像から都道府県を選ぶ

5. 矢印・迷路クイズ
   - スタートからゴールへ到達する命令列を選ぶ
```

これらは魅力的な拡張候補だが、V3 main昇格前のスコープを過大にしないため、Phase 3では対象外とする。

Phase 3では、現行のメニュー構造を変更する。

### 4.1 現行

```text

↓

↓
メニュー
↓
算数レベル選択
↓
クイズ開始
```

現行では、メニュー画面に算数レベル1〜4が直接表示されている。

### 4.2 Phase 3後

```text
タイトル
↓
ユーザー選択
↓
クイズ種別選択
├─ さんすうクイズ
│   ↓
│   算数レベル選択
│   ├─ レベル1
│   ├─ レベル2
│   ├─ レベル3
```

### 4.3 クイズ種別選択

ユーザー選択後、以下の2つを選択できるようにする。

```text

- こっきクイズ
```

### 4.4 さんすうクイズ選択時

「さんすうクイズ」を選択した場合、算数レベル選択画面へ進む。

表示するレベルは以下。

```text
レベル 3（2けたの たし・ひき）
レベル 4（かけざん 九九）
```

「こっきクイズ」を選択した場合、難易度設定画面は表示しない。

```text
こっきクイズ
↓

```

V3 Phase 3では、国旗クイズに難易度設定は設けない。

既存仕様を維持する。

```text
1回のクイズ: 5問
```

### 5.2 選択肢数

既存仕様を維持する。

```text
4択
```

### 5.3 報酬条件

既存仕様を維持する。

```text
5問中4問以上正解で報酬処理
```

算数クイズ、割り算クイズ、国旗クイズのいずれでも、報酬条件は共通とする。

### 5.4 正誤表示

既存仕様を維持する。

```text

- 選択したボタンを correct 表示

不正解時:
- 選択した誤答ボタンを wrong 表示
- 正解ボタンを correct 表示

```

国旗クイズの場合は、正解国名を表示する。

```text
せいかいは 日本
せいかいは フランス
せいかいは ブラジル
```

## 6. クイズデータ構造要求

### 6.1 現行課題

現行の算数クイズは、選択肢と正解が数値前提になっている。

```js
{
  text: "8 + 4 = ?",
  answer: 12,
  choices: [10, 11, 12, 13]
}
```

Phase 3では国旗クイズを扱うため、文字列IDや画像を扱える構造が必要になる。

### 6.2 Phase 3で求める共通構造

Phase 3では、選択肢の表示値と判定値を分離する。

```js
{
  quizId: "math.division.kuku",
  category: "math",
  text: "12 ÷ 3 = ?",
  imageUrl: null,
  answer: 4,
  answerLabel: "4",
  choices: [
    { label: "3", value: 3 },
    { label: "4", value: 4 },
    { label: "5", value: 5 },
    { label: "6", value: 6 }
  ]
}
```

国旗クイズでは以下の形にする。

```js
{
  quizId: "social.flags",
  category: "social",
  text: "この こっきは どこの くに？",
  imageUrl: "https://...",
  answer: "jp",
  answerLabel: "日本",
  choices: [
    { label: "日本", value: "jp" },
    { label: "韓国", value: "kr" },
    { label: "中国", value: "cn" },
    { label: "タイ", value: "th" }
  ]
}
```

### 6.3 要求

```text
1. 算数クイズも国旗クイズも、同じ正誤判定ロジックで処理できること
2. ボタン表示には choice.label を使うこと
3. 正誤判定には choice.value と answer を使うこと
4. 不正解時の正解表示には answerLabel を使うこと
5. answerLabel がない場合は answer を表示できること
6. 国旗クイズでは imageUrl を表示できること
7. 算数クイズでは imageUrl が null でも問題なく動作すること
```

## 9. 割り算クイズ要求仕様

### 9.1 レベル5：九九のわり算

#### 目的

九九の逆算として、余りのない割り算を出題する。

#### 問題例

```text
12 ÷ 3 = ?
24 ÷ 6 = ?
56 ÷ 7 = ?
```

#### 生成ルール

```text
答え: 1〜9
割る数: 1〜9
割られる数: 答え × 割る数
```

#### 要求

```text
1. 必ず割り切れること
2. 答えは1〜9であること
3. 4択で回答できること
4. 既存の報酬仕様に乗ること
5. 不正解時に正解が表示されること
```

### 9.2 レベル6：2けた ÷ 1けた の余りなし割り算

#### 目的

九九のわり算の次段階として、2けたの数を1けたで割る問題を出題する。

#### 問題例

```text
36 ÷ 3 = ?
48 ÷ 4 = ?
84 ÷ 7 = ?
96 ÷ 8 = ?
```

#### 生成ルール

初期版では、難易度を抑える。

```text
割る数: 2〜9
答え: 2〜12
割られる数: 割る数 × 答え
条件: 割られる数が2けたであること
```

#### 要求

```text
1. 必ず割り切れること
2. 割られる数は2けたであること
3. 割る数は1けたであること
4. 初期版では答えは2〜12程度に抑えること
5. 4択で回答できること
6. 既存の報酬仕様に乗ること
7. 不正解時に正解が表示されること
```

## 10. 国旗クイズ要求仕様

### 10.1 目的

国旗を見て、正しい国名を4択で選ぶクイズを追加する。

算数以外のクイズを追加することで、PokeMathの4択クイズ基盤を汎用化する。

### 10.2 出題開始

国旗クイズは、クイズ種別選択で「こっきクイズ」を選んだら、そのまま開始する。

```text
こっきクイズ
↓
問題開始
```

V3 Phase 3では、国旗クイズに難易度設定は設けない。

### 10.3 問題例

```text
この こっきは どこの くに？

1. 日本
2. 韓国
3. 中国
4. タイ
```

### 10.4 表示要求

```text
1. 問題文を表示する
2. 国旗画像を表示する
3. 国名の4択ボタンを表示する
4. 正解時は選択ボタンを correct 表示する
5. 不正解時は誤答ボタンを wrong 表示する
6. 不正解時は正解ボタンを correct 表示する
7. 不正解時は「せいかいは 日本」のように正解国名を表示する
```

### 10.5 出題対象

国旗クイズは、Firestoreの `country_masters` から読み込んだ国・地域マスターを使用する。

```text
出題対象:
country_masters のうち enabled === true のもの
```

## 11. Firestore 国マスター要求仕様

### 11.1 基本方針

国旗クイズ用マスターは、Firestoreに独立して保持する。

```text
country_masters/{countryId}
```

既存のポケモン世代マスター `masters/gen_{1..9}` とは分離する。

理由は以下。

```text
1. 国マスターはポケモン世代マスターとライフサイクルが異なる
2. 将来の都道府県マスターや都道府県形状マスターと独立管理しやすい
3. 国旗クイズの変更が既存ポケモン機能に影響しにくい
4. 将来的に prefecture_masters などを追加しやすい
```

### 11.2 将来拡張時のマスター分離方針

将来拡張では、以下のようにマスターを分離する余地を残す。

```text
country_masters/{countryId}
prefecture_masters/{prefCode}
prefecture_shape_masters/{prefCode}
```

都道府県の形は画像またはSVGパスを扱う可能性があるため、通常の都道府県マスターとは分離する選択肢を残す。

## 12. 国マスター項目要求

### 12.1 ドキュメントID

```text
country_masters/{countryId}
```

`countryId` は、原則として小文字の `cca2` を使用する。

例:

```text
jp
us
fr
br
de
```

### 12.2 項目案

```js
{
  countryId: "jp",
  countryNameJa: "日本",
  countryNameEn: "Japan",

  cca2: "JP",
  cca3: "JPN",

  flagUrl: "https://...",
  flagSvgUrl: "https://...",
  flagPngUrl: "https://...",

  capitalJa: "東京",
  capitalEn: "Tokyo",

  region: "Asia",
  subregion: "Eastern Asia",

  independent: true,
  unMember: true,
  status: "officially-assigned",

  difficulty: null,
  enabled: true,

  source: "REST Countries",
  updatedAt: "2026-06-07"
}
```

### 12.3 difficulty

`difficulty` は `null` とする。

```text
difficulty: null
```

理由は以下。

```text
1. 国旗の難易度判断は主観的になりやすい
2. V3 Phase 3では国旗クイズに難易度設定を設けない
3. 将来的に使う余地だけ残せばよい
```

### 12.4 enabled

`enabled` は出題対象制御に使用する。

```text
enabled === true の国・地域のみ出題対象
```

## 13. 国の定義

### 13.1 採用方針

V3 Phase 3では、REST Countries等から取得できる国・地域データを広く取り込み、Firestoreの `country_masters` に保持する。

```text
採用方針:
REST Countries全件相当を country_masters に登録する
```

ただし、出題対象は `enabled` で制御する。

```text
出題対象:
enabled === true のもの
```

### 13.2 考え方

DBには広く持つ。  
出題は絞れるようにする。

```text
DB登録:
全世界の国・地域データを広く登録

出題:
enabled フラグで制御
```

これにより、以下のような出題制御が将来可能になる。

```text
- 全件出題
- independent=true のみ
- UN加盟国のみ
- UN加盟国 + オブザーバー
- 地域別
```

### 13.3 国の定義比較

| 案 | 定義 | 想定件数 | 特徴 |
| --- | --- | ---: | --- |
| A | REST Countries 全件 | 約250件超 | 国・地域まで含む。国旗クイズとして最も広い |
| B | REST Countries independent=true | 約190〜200件前後 | 「国」としての違和感が少なく、件数も多い |
| C | UN加盟国 | 193件 | 社会科的に説明しやすい |
| D | UN加盟国 + オブザーバー2 | 195件 | バチカン・パレスチナを含められる |
| E | REST Countries全件登録 + enabled制御 | 約250件超登録 | 最も柔軟。今回の採用方針 |

Phase 3では案Eを採用する。

## 14. 国旗クイズの出題ロジック要求

### 14.1 マスター読み込み

国旗クイズ開始時に `country_masters` を読み込む。

要求:

```text
1. enabled === true の国・地域を取得する
2. 初回読み込み後は state にキャッシュする
3. 2回目以降はキャッシュを利用する
4. 読み込み失敗時はエラー表示する
```

### 14.2 問題生成

```text
1. 出題対象から正解国を1件選ぶ
2. 正解以外から誤答候補を3件選ぶ
3. 正解1件 + 誤答3件をシャッフルする
4. 国旗画像URLを imageUrl として問題に持たせる
5. 正解国名を answerLabel に持たせる
```

### 14.3 初期実装の誤答候補

初期実装では、誤答候補は全出題対象からランダムに選ぶ。

将来的には以下を検討できる。

```text
- 同一地域から誤答候補を選ぶ
- 似た国旗グループから選ぶ
- 難易度別に選ぶ
```

V3 Phase 3では実装しない。
