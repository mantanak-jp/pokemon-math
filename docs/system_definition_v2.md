# ポケモンさんすう V2 システム定義書

最終更新: 2026-06-04

## 1. プロジェクト目的

ポケモンさんすう V2 は、算数クイズを解くことで第1〜第9世代のポケモンを順次ゲットし、全世代コンプリートを目指す「世代進行型学習アプリ」です。

## 2. 現在の本番構成

| 項目 | 現在の状態 |
| --- | --- |
| V2 Main 本体 | `index.html` |
| V1.5 退避版 | `archive/index_v1_5.html` |
| V2 Canary 履歴・検証用 | `canary/index.html` |
| V1.5→V2 移行ツール | `canary/migration.html` |
| V2ユーザーデータ | Firestore `users_v2/{userId}` |
| V2マスターデータ | Firestore `masters/gen_{1..9}` |
| V1.5ユーザーデータ | Firestore `users/{username}` |

## 4. ユーザー管理

| userId | displayName | 備考 |
| --- | --- | --- |
| `ryoma` | `りょうま` | V1.5から移行済み。第1世代クリア済みとして第2世代から開始。 |
| `sara` | `さら` | V1.5から移行済み。第1世代途中経過を引き継ぎ。 |
| `guest` | `ゲスト` | 移行対象外。V2検証にも使用。 |

## 5. Firestoreデータ構造

```json
{
  "displayName": "りょうま",
  "cleared_generations": 1,
  "current_gen_owned": []
}
```

`current_gen_owned` には `api_id` ではなく `local_id` を保存します。

## 6. マスターデータ

```text
masters/gen_{1..9}
```

通常プレイ中は読み取り専用です。

## 7. クイズ仕様

- 1回のクイズは5問。
- レベル1〜4を持つ。
- 4問正解で1匹、5問正解で2匹のポケモンを取得します。
- 取得対象は、現在世代の未取得ポケモンです。

## 8. 保存仕様

ポケモン取得直後に Firestore `users_v2/{userId}` へ保存します。

保存時は Firestore transaction を使い、以下を同時に扱います。

- 取得済み `local_id` の追加
- 重複防止
- 世代コンプリート判定
- `cleared_generations` の更新
- `current_gen_owned` のリセット
- 全世代コンプリート判定

## 9. 世代進行仕様

現在世代の取得数が、その世代の `pokemon_list.length` に到達したら世代クリアです。

```json
{
  "cleared_generations": 3,
  "current_gen_owned": []
}
```

第9世代クリア時は `cleared_generations = 9` とし、全世代コンプリートとします。

## 10. 図鑑仕様

図鑑画面では、以下の世代だけを表示します。

- クリア済み世代
- 現在世代

未到達世代は表示しません。

## 11. ポケモン詳細モーダル

- モーダルは `.app` の外、body直下相当の外側レイヤーに置きます。
- モーダル背景は `position: fixed` です。
- 背景クリックで閉じます。
- モーダル本体クリックでは閉じません。
- 右上の × ボタンで閉じます。
- 下部の「とじる」ボタンは表示しません。
- `body.modal-open` は使いません。

## 12. 移行仕様との関係

V1.5→V2移行は `canary/migration.html` で実施しました。

移行対象:

```text
ryoma
sara
```

`guest` は移行対象外です。

## 14. 禁止事項

- V2 Mainから `users` コレクションへ通常プレイ中に書き込まない。
- `masters/gen_{1..9}` へ通常プレイ中に書き込まない。
- `canary/migration.html` への通常導線をMainに置かない。
- `owned_by_generation` のような未採用データ構造を無断で追加しない。
- `body.modal-open` を安易に戻さない。
