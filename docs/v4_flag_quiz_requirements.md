# V4 Flag Quiz Requirements

作成日: 2026-06-11
対象: V4 / 国旗クイズ要求仕様
位置づけ: Step 4 / 要求仕様の確定

## 1. 目的

この文書は、PokeMath V4 に追加する「国旗クイズ」の要求仕様を定義します。

国旗クイズでは、子どもが国旗を見て国名を選ぶ学習体験を提供します。既存の算数クイズと同じアプリ体験・報酬体験に接続しつつ、V3.1 Main と既存ユーザーデータを壊さずに V4 内で段階的に実装します。

## 2. 基本方針

国旗クイズの基本方針は以下です。

```text
- 国旗クイズは v4/ 内で実装する。
- root index.html と v3/ は変更しない。
- メニューで「さんすうクイズ」と「こっきクイズ」を分ける。
- 国旗クイズは算数クイズと同じく 5問構成にする。
- 国旗画像を見て、日本語国名を4択で選ぶ。
- 正解・不正解の表示を行う。
- 不正解時には正解選択肢が分かるようにする。
- 5問終了後に結果画面へ進む。
- 報酬データ構造と保存処理は既存のポケモン報酬と共通にする。
- 報酬条件は算数クイズと国旗クイズで分ける。
- 初期実装では難易度選択なし。
- Step 5 の設計判断により、初期出題は `enabled=true` の全件対象にする。
- 初期から子ども向け代表国だけには絞らない。
- 将来的に地域・難易度・似た国旗グループなどで拡張できる余地を残す。
```

## 3. 今回確定した仕様判断

### 3.1 初期対象国

Step 4 時点では、子ども向けの代表国に絞る方針でした。

Step 5 の設計判断により、方針を更新します。

```text
- country_masters には、可能な範囲で全ての国・地域を登録する。
- 初期出題範囲も、原則として enabled=true の全件対象にする。
- 初期データでは、問題なく出題できる国・地域は enabled=true にする。
- 出題が難しい国・地域だけ enabled=false にする余地を残す。
- 子ども向け代表国への絞り込みは、初期実装では行わない。
- 必要があれば、実装後・実機確認後に enabled / difficulty / similar_group_id で調整する。
```

Step 4 時点で想定していた代表国の例は、将来調整時の参考として扱います。

```text
- 日本
- アメリカ
- イギリス
- フランス
- ドイツ
- イタリア
- スペイン
- ブラジル
- 韓国
- 中国
- インド
- カナダ
- オーストラリア
- など
```

`country_masters` の V4 向け最終設計は `docs/v4_country_master_design.md` で定義します。

### 3.2 難易度

採用案は、初期は難易度なしの方針です。

「こっきクイズ」を選ぶと、すぐ5問の国旗クイズを開始します。初期実装では、難易度選択や地域選択は入れません。

将来的に、難易度、地域、似た国旗グループなどで拡張できるようにします。

### 3.3 報酬

採用案は、既存ポケモン報酬のデータ構造と保存処理を共通利用する方針です。

算数クイズの報酬条件は現行仕様を維持します。国旗クイズは、Step 9-C の実機確認後判断により、国旗クイズ専用の正解数しきい値を使います。国旗クイズ専用の報酬データ構造は作りません。

`users_v2` の既存構造は不用意に変更しません。報酬保存の詳細は、Step 9 実装時に既存処理との整合を確認します。

### 3.4 国旗画像読み込み失敗時

採用案は、可能なら別問題に差し替え、難しければエラー表示とする方針です。

理想は、画像読み込み失敗時にその問題を使わず、別の国旗問題に差し替えることです。初期実装で差し替えが複雑な場合は、エラー表示してメニューへ戻る方針でもよいものとします。

画像なしでそのまま問題を出すことは避けます。

### 3.5 出題選択肢

採用案は、正解1件 + ダミー3件、重複なしの方針です。

各問では、国旗画像を1つ表示します。選択肢は日本語国名4択とし、正解国名を1つ、ダミー国名を3つ含めます。

4択内で国名を重複させません。1セット5問内で、正解国を重複出題しません。

## 4. 画面・導線要求

画面・導線要求は以下です。

```text
- ユーザー選択後のメニューにクイズ種別選択を追加する。
- 「さんすうクイズ」を選ぶと既存のレベル選択に進む。
- 「こっきクイズ」を選ぶと国旗クイズを開始する。
- 既存の算数クイズ導線を壊さない。
- 国旗クイズ中は、国旗画像、問題番号、正解数、4つの国名選択肢を表示する。
- 回答後は正解・不正解を表示する。
- 不正解時は正しい国名が分かるようにする。
- 5問後に結果画面へ進む。
- 結果画面からメニューに戻れる。
- 必要に応じて図鑑へ進める。
- iPhone表示を優先する。
```

## 5. 出題仕様

出題仕様は以下です。

```text
- 1セット5問。
- 各問は国旗画像を1つ表示し、選択肢として日本語国名を4つ表示する。
- 正解は1つ。
- ダミー選択肢は3つ。
- 選択肢は country_masters から生成する。
- 同一セット内で同じ正解国を重複出題しない。
- 選択肢内で同じ国名を重複させない。
- 初期実装では、enabled=true の国・地域全件からランダム出題する。
- 初期から子ども向け代表国だけには絞らない。
- 将来的には地域、難易度、similar_group_id による制御を検討する。
- 国旗画像が読み込めない場合は、可能なら別問題に差し替える。
- 差し替えが難しい場合は、エラー表示してメニューへ戻る。
- 画像なしでそのまま問題を出すことは避ける。
```

## 6. 判定仕様

判定仕様は以下です。

```text
- ユーザーが選択肢を押した時点で回答確定。
- 正解なら正解表示。
- 不正解なら選択した選択肢を不正解表示し、正解選択肢を正解表示する。
- 回答後、一定時間後に次の問題へ進む。
- 算数クイズの現行仕様に近いテンポにする。
- 5問終了後、結果画面へ進む。
```

## 7. 報酬仕様

報酬仕様は以下です。

```text
- 算数クイズは既存仕様を維持する。
- 算数クイズ: 0〜3問正解で0匹、4問正解で1匹、5問正解で2匹。
- 国旗クイズ: 0〜2問正解で0匹、3〜4問正解で1匹、5問正解で2匹。
- 報酬データ構造と保存処理は既存のポケモン報酬仕様と共通にする。
- 国旗クイズ専用の報酬データ構造は初期実装では作らない。
- users_v2 の既存構造を不用意に変更しない。
- country_masters は読み取り専用とする。
```

## 8. データ要求

国旗クイズは `country_masters` を参照する想定です。

`country_masters` の最終スキーマは Step 5 で `docs/v4_country_master_design.md` に定義します。V3.2 / Phase 4 の先行設計として作成済みの `docs/v3_country_master_design.md` は参考資料として維持します。

基本項目案は以下です。

```text
- country_id
- country_name_ja
- country_name_en
- flag_url
- region
- capital_ja
- difficulty
- similar_group_id
- enabled
- source
- updated_at
```

国コードは ISO 3166-1 alpha-2 を第一候補とします。

`flag_url` の出典・形式は Step 5 で確定します。`enabled=true` の初期対象国・地域は Step 5 または Step 6 で確定します。

## 9. 非機能要求・安全要求

非機能要求・安全要求は以下です。

```text
- V3.1 Main を壊さない。
- root index.html は変更しない。
- v3/ は変更しない。
- Firestore実データに不用意に書き込まない。
- users_v2 の既存構造を壊さない。
- masters/gen_{1..9} は変更しない。
- iPhone表示で破綻しない。
- 国旗画像読み込み失敗時の最低限の扱いを考慮する。
- 国旗画像の外部参照に依存する場合は、読み込み失敗リスクを明記する。
```

## 11. 後続ステップとの関係

後続ステップは以下です。

```text
Step 5: country_masters設計のV4向け確定
Step 6: country_masters.generated.json 作成
Step 7: Firestore投入ツール作成
Step 8: Firestore country_masters 投入・確認
Step 9-A: V4国旗クイズの最小実装
Step 9-B: 報酬接続・結果画面調整・実機確認調整
Step 10: V4実機確認・安定化
```

Step 9-A では、メニューから「こっきクイズ」を選び、Firestore `country_masters` の `enabled=true` から5問を生成し、国旗画像と日本語国名4択で回答できるところまでを実装します。

Step 9-B では、国旗クイズ結果を既存のポケモン報酬保存フローへ接続しました。

Step 9-C では、実機確認後の判断により、国旗クイズのみ 0〜2問正解で0匹、3〜4問正解で1匹、5問正解で2匹に変更します。算数クイズの報酬条件は変更しません。
