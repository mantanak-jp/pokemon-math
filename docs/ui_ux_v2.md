# V2 UI/UX仕様

最終更新: 2026-06-02

## 1. 目的

この文書は、ポケモンさんすうアプリ V2 の UI/UX 方針を定義します。

対象は、通常プレイ中の子ども向け画面です。

V1 から V2 への移行UIは、通常ユーザーには表示しません。移行UIの詳細は `docs/migration_v1_to_v2.md` で扱います。

## 2. 前提

この文書は、以下のドキュメントを前提とします。

- `docs/current_inventory.md`
- `docs/data_model_v1_v2.md`
- `docs/migration_v1_to_v2.md`
- `docs/system_definition_v1.5.md`
- `docs/system_definition_v2.md`

特に、V2 のデータ構造は `docs/data_model_v1_v2.md` を正本とします。

## 3. V2 UI/UX 基本方針

- 子どもが迷わず操作できる画面数にします。
- 世代情報は独立した世代選択画面ではなく、メニュー画面や図鑑画面の中で表示します。
- 取得数、残り数、保存状態を分かりやすく表示します。
- 未取得ポケモンは隠し、集める楽しさを維持します。
- 保存失敗時は失敗を隠さず、再試行できるようにします。
- 重要操作では、誤タップ・連打・二重実行を防止します。
- 文言は、ひらがな + カタカナ中心にします。

## 4. 画面構成

V2 の基本画面構成は以下です。

```text
タイトル画面
↓
ユーザー選択画面
↓
メニュー画面
├─ さんすうを はじめる
│  ↓
│  クイズ画面
│  ↓
│  結果画面
│  ↓
│  必要に応じて 世代クリア / 次世代スタート画面
│
└─ ずかんを みる
   ↓
   図鑑画面
   ↓
   ポケモン詳細モーダル
```

世代選択画面は独立させません。

世代情報は、メニュー画面、クイズ画面、結果画面、図鑑画面の中で表示します。

## 5. ユーザー選択画面

ユーザー選択画面では、以下の3つのボタンのみを表示します。

```text
りょうま
さら
ゲスト
```

ユーザー選択画面では、進行世代、取得数、残り数などの詳細情報は表示しません。

理由:

- 入口画面をシンプルにするためです。
- 詳細な進行状況は、ユーザー選択後のメニュー画面で表示します。

## 6. メニュー画面

メニュー画面では、以下を表示します。

- 表示名
- 現在世代
- 現在世代の取得数
- 次世代までの残り数
- `さんすうを はじめる` ボタン
- `ずかんを みる` ボタン

表示例:

```text
りょうま

いまは 第2せだい（ジョウト）

23 / 100 ひき ゲット！
あと 77ひきで クリア！

[さんすうを はじめる]
[ずかんを みる]
```

## 7. 世代表示ルール

世代表示は、以下の形式を基本とします。

```text
第1せだい（カントー）
第2せだい（ジョウト）
第3せだい（ホウエン）
```

第1〜第9世代の表示名は以下です。

| 世代 | 表示名 |
| --- | --- |
| 1 | 第1せだい（カントー） |
| 2 | 第2せだい（ジョウト） |
| 3 | 第3せだい（ホウエン） |
| 4 | 第4せだい（シンオウ） |
| 5 | 第5せだい（イッシュ） |
| 6 | 第6せだい（カロス） |
| 7 | 第7せだい（アローラ） |
| 8 | 第8せだい（ガラル） |
| 9 | 第9せだい（パルデア） |

注記:

- 文言は子ども向けに `せだい` をひらがな表記にします。
- 地方名はカタカナ表記にします。

## 8. クイズ画面

クイズ画面では、現在世代を小さく表示します。

表示例:

```text
第2せだい（ジョウト）

3 + 4 = ?

[5] [6] [7] [8]
```

クイズ画面では、世代情報を大きく出しすぎず、主役は算数問題にします。

## 9. 結果画面

結果画面では、以下を表示します。

- 正解数
- ゲットしたポケモン
- 保存状態
- 次世代までの残り数

表示例:

```text
5もんちゅう 4もん せいかい！

ピカチュウを ゲット！

ほぞんしています...
↓
ほぞんできました！

あと 12ひきで クリア！
```

全世代コンプリート後にクイズを継続した場合、新規ポケモン取得は発生しません。

その場合は、以下のように表示します。

```text
5もんちゅう 4もん せいかい！

もう ぜんぶ ゲットしているよ！
```

## 10. 保存状態表示

保存状態は、結果画面で明確に表示します。

### 10.1 保存中

```text
ほぞんしています...
```

### 10.2 保存成功

```text
ほぞんできました！
```

### 10.3 保存失敗

保存失敗時は、以下の表記にします。

```text
ゲットできませんでした
```

保存失敗を隠して、ゲット済みのように見せてはいけません。

## 11. 保存失敗時の扱い

保存失敗時は、以下の方針とします。

- 自動再試行を行います。
- 自動再試行でも失敗した場合、手動再試行ボタンを表示します。
- 手動再試行ボタンの文言は `もういちどゲットする` とします。

表示例:

```text
ゲットできませんでした

[もういちどゲットする]
```

実装時は、同じポケモンを二重に追加しないようにします。

## 12. 図鑑画面

図鑑画面では、以下の世代だけを表示します。

- クリア済み世代
- 現在世代

未到達世代は表示しません。

表示例:

```text
[第1せだい（カントー）] [第2せだい（ジョウト）]

第1せだい（カントー）
151 / 151

第2せだい（ジョウト）
23 / 100
```

## 13. クリア済み世代の図鑑表示

クリア済み世代は、全ポケモンを取得済みとして表示します。

理由:

- データモデル上、クリア済み世代の個別取得履歴は保存しないためです。
- `cleared_generations` により、クリア済み世代であることを判断できます。
- クリア済み世代はコンプリート済みとして扱う方針です。

## 14. 現在世代の未取得表示

現在世代の未取得ポケモンは、V1.5 と同様に隠します。

表示方針:

```text
画像: ?
名前: ???
```

未取得ポケモンをタップした場合は、以下を表示します。

```text
まだ つかまえていないよ
```

## 15. ポケモン詳細モーダル

取得済みポケモンをタップした場合、詳細モーダルを表示します。

表示項目は以下です。

- 名前
- 画像
- タイプ
- 高さ
- 重さ
- flavor

全国図鑑IDや世代内IDは、子ども向け画面では原則表示しません。

`flavor` は Firestore に保存済みの値をそのまま表示します。

ただし、表示時の軽微な整形は許容します。

- 折り返し
- 表示幅に合わせたレイアウト調整
- 前後の不要な空白の除去

## 16. 世代開始画面

現在世代の `current_gen_owned` が0匹の場合、その世代のスタート画面を表示します。

表示例:

```text
ジョウトちほうへ！

第2せだいの ポケモンを
ゲットしにいこう！

[さんすうを はじめる]
[メニューにもどる]
```

### 16.1 V1から151匹コンプリート済みで移行されたユーザー

V1で151匹コンプリート済みのユーザーは、移行後に以下の状態になります。

```json
{
  "cleared_generations": 1,
  "current_gen_owned": []
}
```

この状態は、第1世代クリア済み、第2世代開始直後を意味します。

そのため、V2初回利用時には第2世代のスタート画面を表示します。

### 16.2 表示済み管理

世代スタート画面の表示済み管理は、Firestoreではなく `sessionStorage` を使用します。

例:

```text
shown_start_screen_ryoma_gen_2
```

同一ブラウザセッション中は、同じユーザー・同じ世代について1回だけ表示します。

## 17. 世代クリア・次世代解放演出

現在世代をコンプリートした場合、次世代解放画面を表示します。

表示例:

```text
第1せだい クリア！

カントーちほうの ポケモンを
ぜんぶ ゲットしたよ！

[つぎの せだいへ]
```

`つぎの せだいへ` ボタンを押すことで、次世代へ進んだ感覚を出します。

データ上は、以下のように更新します。

```json
{
  "cleared_generations": 1,
  "current_gen_owned": []
}
```

次世代に進んだ後、その世代のスタート画面を表示します。

## 18. 全世代コンプリート後の扱い

第9世代まで完了した場合、全世代コンプリート画面を表示します。

表示例:

```text
ぜんせだい コンプリート！

すべての ポケモンを
ゲットしたよ！

[さんすうを つづける]
[ずかんを みる]
```

全世代コンプリート後は、図鑑を全開放します。

ただし、クイズは継続可能にします。

クイズ継続時は新規ポケモン取得は発生しません。

## 19. 子ども向け文言方針

文言は、ひらがな + カタカナ中心にします。

方針:

- ボタン文言はなるべくひらがな中心にします。
- ポケモン名、地方名、タイプ名はカタカナ・ひらがな表記を許容します。
- 漢字は必要最小限にします。
- `第1せだい` のように、数字とひらがなを組み合わせる表現は許容します。

例:

| 用途 | 文言 |
| --- | --- |
| クイズ開始 | `さんすうを はじめる` |
| 図鑑 | `ずかんを みる` |
| 次世代 | `つぎの せだいへ` |
| 保存中 | `ほぞんしています...` |
| 保存成功 | `ほぞんできました！` |
| 保存失敗 | `ゲットできませんでした` |
| 再試行 | `もういちどゲットする` |

## 20. 誤タップ・連打対策

重要操作では、誤タップ・連打・二重実行を防止します。

対象例:

- クイズ回答ボタン
- ポケモン取得処理
- 保存処理
- `もういちどゲットする` ボタン
- `つぎの せだいへ` ボタン
- Canary移行ボタン

対策:

- ボタン押下後に `disabled` にします。
- 保存中は必要に応じて画面操作を制限します。
- 処理中であることを画面に表示します。
- 二重保存・二重取得を避けるため、処理中フラグを持ちます。

## 21. Canary移行UIとの関係

Canary専用の移行UIは、この文書では詳細化しません。

移行UIの仕様は `docs/migration_v1_to_v2.md` で扱います。

この文書では、以下のみを前提とします。

- 通常ユーザーには移行UIを表示しません。
- Mainには移行UIを表示しません。
- Canaryのゲスト画面にのみ運用者向け移行UIを表示します。
