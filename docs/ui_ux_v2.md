# V2 UI/UX仕様

最終更新: 2026-06-04

## 1. 目的

この文書は、ポケモンさんすうアプリ V2 Main の UI/UX 方針を定義します。

対象は、通常プレイ中の子ども向け画面です。

V1.5 から V2 への移行UIは、通常ユーザーには表示しません。移行UIの詳細は `docs/migration_v1_to_v2.md` で扱います。

## 2. V2 UI/UX 基本方針

- 子どもが迷わず操作できる画面数にする。
- 世代情報は独立した世代選択画面ではなく、メニュー画面・クイズ画面・図鑑画面の中で表示する。
- 現在世代、取得数、保存状態を分かりやすく表示する。
- 未取得ポケモンは隠し、集める楽しさを維持する。
- 保存失敗時は失敗を隠さず、再試行できるようにする。
- 通常結果画面から図鑑へ移動しやすくする。
- 世代クリア時は、クリアと次世代解放を1画面で分かりやすく伝える。
- 文言は子ども向けにやさしくするが、世代表記は「第◯世代」と漢字表記に統一する。

## 3. 画面構成

```text
タイトル画面
↓
ユーザー選択画面
↓
メニュー画面
├─ レベル選択
│  ↓
│  必要に応じて 世代開始画面
│  ↓
│  クイズ画面
│  ↓
│  結果画面
│  ├─ 通常時: 同じレベルでつづける / ずかんをみる / メニューへもどる
│  └─ 世代クリア時: 第◯世代クリア！ → 世代クリア・次世代解放 統合画面
│
└─ ずかんをみる
   ↓
   図鑑画面
   ↓
   ポケモン詳細モーダル
```

## 4. 結果画面

通常のポケモン取得後のボタン構成は以下です。

```text
[同じレベルでつづける]
[ずかんをみる]
[メニューへもどる]
```

保存失敗時の手動再試行ボタンは以下です。

```text
[もういちどゲットする]
```

## 5. 世代クリア画面

世代クリアが発生した場合、結果画面では以下のみを主導線にします。

```text
[第◯世代クリア！]
```

その後、世代クリア・次世代解放の統合画面を表示します。

```text
[同じレベルでつづける]
[ずかんをみる]
[メニューへもどる]
```

## 6. 全世代コンプリート

第9世代をクリアした場合、全世代コンプリート画面を表示します。

全世代コンプリート後もクイズは継続可能です。

ただし、以下を守ります。

- 新規ポケモン取得は発生しない。
- Firestore 書き込みも発生しない。
- 図鑑は全世代を全開放する。

## 7. 図鑑画面

図鑑画面では、以下の世代だけを表示します。

- クリア済み世代
- 現在世代

未到達世代は表示しません。

未取得ポケモンは、以下のように隠します。

```text
?
???
```

## 8. 図鑑グリッドのスクロール仕様

図鑑画面全体を長いスクロール画面にしません。

図鑑グリッド部分だけをスクロール領域にします。

`.zukan-grid` の高さ指定は以下です。

```css
max-height: min(58vh, 520px);
max-height: min(58dvh, 520px);
```

## 9. ポケモン詳細モーダル

- モーダルは .app の外、body直下相当の外側レイヤーに置く。
- モーダル背景は position: fixed。
- 背景クリックで閉じる。
- モーダル本体クリックでは閉じない。
- 右上の × ボタンで閉じる。
- 下部の「とじる」ボタンは表示しない。
- body.modal-open は使わない。

## 10. ユーザー別の初期表示確認済み状態

```text
りょうま:
第2世代（ジョウト）
0 / 100 ひき ゲットずみ

さら:
第1世代（カントー）
115 / 151 ひき ゲットずみ

ゲスト:
移行対象外。
V2検証にも使用した users_v2/guest データ。
```

`guest` はV1.5からの移行対象外ですが、V2 Main上では通常ユーザー選択肢として表示されます。
