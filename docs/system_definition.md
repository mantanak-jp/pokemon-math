# ポケモンさんすう V2 システム定義書

## 1. プロジェクト目的
算数クイズを通じて1〜9世代のポケモンを順次ゲットし、全世代コンプリートを目指す「世代進行型学習アプリ」への刷新。

## 2. 機能要件
* **世代進行サイクル**: 現在の到達世代コンプリートで次世代が自動解放される。
* **図鑑閲覧**: 到達世代までのデータをタブで切替表示。未来の世代は非表示。
* **データ継続性**: 既存の `owned` データを維持し、`current_gen` を拡張することで進行を管理する。

## 3. 詳細設計
### データ構造 (Firestore)
* `masters/gen_{n}`: 世代別マスターデータ（既構築）。
* `users/{userId}`:
    * `owned`: 保有ポケモンID配列（既存）。
    * `current_gen`: 現在挑戦中の世代（数値、初期値1）。

### ロジックフロー
1. **初期化**: `current_gen` が無ければ `1` を補完（既存ユーザー対策）。
2. **クイズ生成**: `current_gen` 世代のマスターデータを参照し、`owned` にないポケモンを出題。
3. **進行判定**: 全所持確認後 `current_gen` をインクリメントして保存。

## 4. 進捗状況 (Step 4)
* Step 1-3: UI統合・マスター構築・タブ図鑑完了。
* Step 4: 進行度管理（current_gen）の導入。
* Step 5: V2クイズロジックへの置換（未）。
