# ポケモンさんすうアプリ v1.5 仕様定義書

## 1. アプリケーション概要
本アプリケーションは、子どもが楽しく算数の学習を継続できるよう、クイズの成果に応じて「ポケットモンスター（初代151匹）」を収集・図鑑登録できるモバイルファーストのWebアプリケーション（SPA: Single Page Application）である。

---

## 2. 機能要件 (Functional Requirements)

### 2.1. 画面構成および遷移
アプリケーションは単一のHTML内で画面表示を切り替えるCSSクラス制御（`.screen.active`）を採用しており、以下の6画面で構成される。

1. **タイトル画面 (`screen-title`)**
   * アプリ起動時の初期画面。データ読み込み完了まで「じゅんびちゅう...」と表示し、完了後に「はじめる」ボタンを活性化する。
2. **プレイヤー選択画面 (`screen-user`)**
   * プレイするユーザーを「りょうま」「さら」「ゲスト」の3つから選択する。
3. **メインメニュー画面 (`screen-menu`)**
   * 4つのクイズレベル選択、および「ポケモンずかんを みる」「プレイヤーを かえる」への導線を提供する。
4. **クイズ画面 (`screen-quiz`)**
   * 算数クイズを1セッションあたり5問出題する。
5. **結果発表画面 (`screen-result`)**
   * 正解数を表示し、成果に応じたポケモン獲得アニメーションおよびメッセージを表示する。
6. **ポケモン図鑑画面 (`screen-zukan`)**
   * 現在選択されているユーザーのポケモン獲得状況（取得率および詳細情報）を確認できる。

### 2.2. ユーザー管理機能
* ユーザーごとに独立したポケモン所持データ（`owned`リスト）を保持する。
* ユーザー選択時に、Firebase Firestoreから該当ユーザーのデータを非同期でロードする。
* 「ゲスト」ユーザーも他ユーザーと同様に独立したデータ枠を持つ。

### 2.3. クイズ出題機能
* 1セッションにつき**全5問**の4択クイズが出題される。
* 上部に問題数カウント（例：だい 1 もん）、現在セッションの正解数、および視覚的な進捗バー（プログレスバー）を表示する。
* **解答時のフィードバック**:
  * 選択肢をクリックした瞬間、正誤判定を即座に行う（多重解答を防ぐロック判定あり）。
  * 正解時はボタンが**緑色**になり、「⭕ せいかい！」と表示。
  * 不正解時はクリックしたボタンが**赤色**になり、「❌ ざんねん！」と表示。
  * 判定表示から1.2秒後に自動で次の問題または結果画面へ遷移する。

### 2.4. 報酬（ポケモンゲット）システム
クイズの正解数（5問中）に応じて、以下の報酬が発生する。

* **4問以上正解 (4/5 または 5/5)**: ポケモンをゲットできる。
  * **5問全問正解**: **2匹**のポケモンを獲得（演出：✨⭐🔴⭐✨）。
  * **4問正解**: **1匹**のポケモンを獲得（演出：🔴）。
  * **出現プール制御**: 選択したクイズのレベルに応じて、出現するポケモンの範囲（図鑑No.）が制限される。
  * **重複回避**: 出現プール内の「未所持ポケモン」からランダムに抽選される。万が一、該当レベルのプールをコンプリートした場合は、151匹の全プールから未所持が再抽選される。
  * **データ保存**: 獲得に成功した場合、Firebase Firestoreへ即座に自動保存される。
* **3問以下正解**: ポケモンは逃げてしまう（演出：💨、「おしい！もういちど ちょうせん しよう！」を表示）。

### 2.5. ポケモン図鑑・モーダル機能
* 初代151匹のグリッド表示。
* **未所持のポケモン**: 図鑑No.のみ表示され、画像は「?」、名前は「???」とマスクされる。
* **所持しているポケモン**: 公式アートワークの画像と名前が表示され、クリック可能となる。
* **詳細モーダル（ポップアップ）**: 所持済みのポケモンをクリックすると、以下の詳細データがポップアップ表示される。
  * 図鑑No.、ポケモン名、タイプ（バッジ表示）、高さ、重さ、解説文（ひらがな・カタカナ優先）。

## 3. 詳細設計 (Detailed Design)

### 3.1. クイズ生成ロジック (レベル別仕様)
各レベルにおける計算式の変数（$n_1$, $n_2$）および演算子（$op$）の決定アルゴリズムは以下の通り。

| レベル | 内容 | 算出ロジック | 備考 |
| :--- | :--- | :--- | :--- |
| **レベル 1** | 1けたの たし・ひき | $n_1$: 1〜9 の乱数<br>$n_2$: 1〜9 の乱数<br>50%の確率で「$-$」に設定。 | 引き算の場合、答えがマイナスにならないよう $n_1 \ge n_2$ の場合のみ「$-$」とする。 |
| **レベル 2** | 2けた と 1けた | $n_1$: 10〜29 の乱数<br>$n_2$: 1〜9 の乱数<br>50%の確率で「$-$」に設定。 | 引き算のマイナス制限はなし。 |
| **レベル 3** | 2けた の たし・ひき | $n_1$: 20〜69 の乱数<br>$n_2$: 10〜49 の乱数<br>50%の確率で「$-$」に設定。 | 引き算の場合、答えがマイナスにならないよう $n_1 \ge n_2$ の場合のみ「$-$」とする。 |
| **レベル 4** | かけざん 九九 | $n_1$: 1〜9 の乱数<br>$n_2$: 1〜9 の乱数<br>演算子は「$\times$」固定。 | 九九の範囲からランダム出題。 |

#### 選択肢（4択）の自動生成
1. 正解の値を最初の要素とする。
2. 残り3つの選択肢は、`正解の値 + (-4 〜 +4 のランダムな整数)` で生成する。
3. ただし、値が 0以上 かつ 重複しない ことを条件とし、4つのユニークな選択肢が揃うまでループする。
4. 最終的に配列をランダムにシャッフルしてUI（ボタン）へ配置する。

### 3.2. レベル別ポケモン出現プール仕様
レベルごとの難易度に見合った達成感を与えるため、獲得できるポケモンの図鑑No.の上限が制御されている。

* **レベル 1 & レベル 2**: 図鑑No.1 〜 **100** まで
* **レベル 3**: 図鑑No.1 〜 **143** まで
* **レベル 4**: 図鑑No.1 〜 **151**（伝説・幻のポケモンを含む全種類）

### 3.3. 外部データ・API連携設計

#### 1. PokeAPI (`https://pokeapi.co/api/v2/`)
* **通信タイミング**: アプリ起動時（`window.onload`）に、No.1からNo.151までのデータを `Promise.all` を用いて非同期・並行処理で一括取得（キャッシュ）する。
* **データ整形**:
  * `detail`エンドポイントおよび`pokemon-species`エンドポイントの2つからデータを統合する。
  * **名前 (`name`)**: `ja-Hrkt`（ひらがな・カタカナ）または `ja` のローカライズ名を優先取得。
  * **解説文 (`flavor`)**: 子どもが読めるよう、`ja-Hrkt`（ひらがな・カタカナ版）のエントリーを最優先で探索し、改行やフォームフィード文字を半角スペースに一括置換して格納する。
  * **単位変換**: API仕様に合わせ、高さは 1/10（メートル換算）、重さは 1/10（キログラム換算）に変換。

#### 2. Firebase Firestore (v10.8.0 compat)
* **コレクション構造**: `users/{username}`
* **ドキュメント内データ構造**:
  ```json
  {
    "owned": [1, 4, 7, 25], 
    "updatedAt": "Timestamp"
  }
