# V2 UI/UX仕様

最終更新: 2026-06-04

## 1. 目的

この文書は、ポケモンさんすうアプリ V2 の UI/UX 方針を定義します。

対象は、通常プレイ中の子ども向け画面です。

V1.5 から V2 への移行UIは、通常ユーザーには表示しません。移行UIの詳細は `docs/migration_v1_to_v2.md` で扱います。

---

## 2. 前提

この文書は、以下のドキュメントを前提とします。

```text
docs/current_inventory.md
docs/data_model_v1_v2.md
docs/migration_v1_to_v2.md
docs/system_definition_v1.5.md
docs/system_definition_v2.md

AGENTS.md
```

特に、V2 のデータ構造は `docs/data_model_v1_v2.md` を正本とします。

## 3. V2 UI/UX 基本方針

```text
- 子どもが迷わず操作できる画面数にする。
- 世代情報は独立した世代選択画面ではなく、メニュー画面・クイズ画面・図鑑画面の中で表示する。
- 現在世代、取得数、保存状態を分かりやすく表示する。
- 未取得ポケモンは隠し、集める楽しさを維持する。
- 保存失敗時は失敗を隠さず、再試行できるようにする。
- 重要操作では、誤タップ・連打・二重実行を防止する。
- 通常結果画面から図鑑へ移動しやすくする。
- 世代クリア時は、クリアと次世代解放を1画面で分かりやすく伝える。
- 文言は子ども向けにやさしくするが、世代表記は「第◯世代」と漢字表記に統一する。
```

## 4. 画面構成

V2 の基本画面構成は以下です。

```text
タイトル画面
↓
ユーザー選択画面
↓
メニュー画面
├─ レベル選択
│  ↓
│  必要に応じて 世代開始画面
│  ↓
│  クイズ画面
│  ↓
│  結果画面
│  ├─ 通常時: 同じレベルでつづける / ずかんをみる / メニューへもどる
│  └─ 世代クリア時: 第◯世代クリア！ → 世代クリア・次世代解放 統合画面
│
└─ ずかんをみる
   ↓
   図鑑画面
   ↓
   ポケモン詳細モーダル
```

世代選択画面は独立させません。

世代情報は、メニュー画面、クイズ画面、結果画面、図鑑画面の中で表示します。

---

## 5. タイトル画面

タイトル画面では、以下を表示します。

```text
ポケモンさんすう

世代を すすめる あたらしい ぼうけん

[はじめる]
```

タイトルは以下に統一します。

```text
ポケモンさんすう
```

タイトルは改行しません。

サブタイトルは以下に統一します。

```text
世代を すすめる あたらしい ぼうけん
```

トップ画面にポケモン画像を出す場合、ポケモン名キャプションは表示しません。

---

## 6. ユーザー選択画面

ユーザー選択画面では、以下の3つのボタンを表示します。

```text
りょうま
さら
ゲスト
```

ユーザー選択画面では、進行世代、取得数、残り数などの詳細情報は表示しません。

理由:

```text
- 入口画面をシンプルにするため。
- 詳細な進行状況は、ユーザー選択後のメニュー画面で表示するため。
```

ユーザー選択画面にポケモン画像を出す場合、ポケモン名キャプションは表示しません。

---

## 7. メニュー画面

メニュー画面では、以下を表示します。

```text
- 表示名
- 現在世代
- 現在世代の取得数
- レベル選択ボタン
- ずかんをみる
- ユーザーをかえる
```

現在の実装では、独立した `さんすうを はじめる` ボタンではなく、4つのレベルボタンを直接表示します。

表示例:

```text
✨ りょうま の メニュー ✨

第2世代（ジョウト）  0ひき ゲットずみ！

[レベル 1（1けたの たし・ひき）]
[レベル 2（2けた と 1けた）]
[レベル 3（2けたの たし・ひき）]
[レベル 4（かけざん 九九）]

[📕 ずかんを みる]
[👤 ユーザーを かえる]
```

全世代コンプリート後は以下のように表示します。

```text
全世代コンプリート！
さんすうに ちょうせんできるよ！
```

## 8. 世代表示ルール

世代表示は、以下の形式を基本とします。

```text
第1世代（カントー）
第2世代（ジョウト）
第3世代（ホウエン）
```

第1〜第9世代の表示名は以下です。

| 世代 | 表示名        |
| -- | ---------- |
| 1  | 第1世代（カントー） |
| 2  | 第2世代（ジョウト） |
| 3  | 第3世代（ホウエン） |
| 4  | 第4世代（シンオウ） |
| 5  | 第5世代（イッシュ） |
| 6  | 第6世代（カロス）  |
| 7  | 第7世代（アローラ） |
| 8  | 第8世代（ガラル）  |
| 9  | 第9世代（パルデア） |

方針:

```text
- 「第◯世代」を使う。
- 「第◯せだい」には戻さない。
- 地方名はカタカナ表記にする。
```

## 9. 世代開始画面

現在世代の `current_gen_owned` が0匹の場合、その世代の開始画面を表示します。

ただし、表示するのは以下の条件を満たす場合です。

```text
- current_gen_owned が0匹
- 同一セッションで、そのユーザー・その世代の開始画面をまだ表示していない
- ユーザーがメニュー画面でレベルを選択した
```

表示例:

```text
ジョウトちほうへ！

第2世代の ポケモンを
ゲットしにいこう！

[このレベルで はじめる]
[メニューへ もどる]
```

表示済み管理は Firestore ではなく `sessionStorage` を使います。

キー例:

```text
v2_start_screen_shown_ryoma_gen_2
```

世代クリア直後には、この世代開始画面は挟みません。
世代クリア直後は、後述の「世代クリア・次世代解放 統合画面」から直接同じレベルでクイズを開始します。

---

## 10. クイズ画面

クイズ画面では、現在世代を小さく表示します。

表示例:

```text
第2世代（ジョウト）

だい 1 もん
せいかい: 0

3 + 4 = ?

[5] [6] [7] [8]
```

クイズ画面では、世代情報を大きく出しすぎず、主役は算数問題にします。

### 10.1 問題数

1回のクイズは5問です。

```text
QUIZ_QUESTION_COUNT = 5
```

### 10.2 レベル

```text
レベル 1: 1けたの たし・ひき
レベル 2: 2けた と 1けた
レベル 3: 2けたの たし・ひき
レベル 4: かけざん 九九
```

---

## 11. ポケモン取得ルール

ポケモン取得数は正解数で決まります。

```text
0〜3問正解: 0匹
4問正解: 1匹
5問正解: 2匹
```

取得対象は、現在世代の未取得ポケモンです。

保存時は `users_v2/{userId}.current_gen_owned` に `local_id` を追加します。

世代クリア済みの世代について、個別取得履歴は保存しません。
クリア済み世代は `cleared_generations` によって全取得済みとして扱います。

---

## 12. 結果画面

結果画面では、以下を表示します。

```text
- 正解数
- ゲットしたポケモン
- 保存状態
- 次の行動ボタン
```

通常のポケモン取得後のボタン構成は以下です。

```text
[同じレベルでつづける]
[ずかんをみる]
[メニューへもどる]
```

「もういちど」は通常結果画面では使いません。
通常結果画面では、意味が明確な以下の文言を使います。

```text
同じレベルでつづける
```

理由:

```text
子どもが「同じレベルでもう1回計算する」ことを理解しやすくするため。
```

また、子どもの遊び方として、計算後に図鑑へ移動することが多いため、結果画面に以下の導線を追加します。

```text
ずかんをみる
```

---

## 13. 4問未満の場合

4問未満でポケモンを取得できなかった場合も、結果画面には以下の3ボタンを表示します。

```text
[同じレベルでつづける]
[ずかんをみる]
[メニューへもどる]
```

表示例:

```text
5もん中 3もん せいかい！

おしい！
同じレベルで つづけてみよう！

[同じレベルでつづける]
[ずかんをみる]
[メニューへもどる]
```

---

## 14. 保存状態表示

保存状態は、結果画面で明確に表示します。

### 14.1 保存中

```text
ポケモンを ゲットしているよ...
```

### 14.2 保存成功

```text
ほぞんできました！
```

通常取得時の表示例:

```text
5もん中 4もん せいかい！

ピカチュウを ゲット！

ほぞんできました！

[同じレベルでつづける]
[ずかんをみる]
[メニューへもどる]
```

### 14.3 保存失敗

保存失敗時は、以下の表記にします。

```text
ゲットできませんでした
```

保存失敗を隠して、ゲット済みのように見せてはいけません。

---

## 15. 保存失敗時の扱い

保存失敗時は、以下の方針とします。

```text
- 自動再試行を行う。
- 自動再試行でも失敗した場合、手動再試行ボタンを表示する。
- 手動再試行ボタンの文言は「もういちどゲットする」とする。
```

表示例:

```text
ゲットできませんでした

[もういちどゲットする]
[メニューへもどる]
```

重要:

```text
- 通常結果画面の「同じレベルでつづける」とは別扱い。
- 保存失敗時の「もういちどゲットする」は維持する。
- 同じポケモンを二重に追加しないようにする。
```

---

## 16. 世代クリア時の結果画面

世代クリアが発生した場合、結果画面では以下を表示します。

```text
- 正解数
- ゲットしたポケモン
- 保存成功メッセージ
- この世代のポケモンを ぜんぶ ゲットしたよ！
- 第◯世代クリア！ ボタン
```

表示例:

```text
5もん中 4もん せいかい！

ラグラージを ゲット！

ほぞんできました！
この世代のポケモンを ぜんぶ ゲットしたよ！

[第3世代クリア！]
```

この画面では、以下の通常ボタンは表示しません。

```text
- 同じレベルでつづける
- ずかんをみる
- メニューへもどる
```

理由:

```text
世代クリア時は、まずクリア後の統合画面へ進ませる方が自然なため。
```

---

## 17. 世代クリア・次世代解放 統合画面

世代クリア後は、世代クリア画面と次世代スタート画面を分けず、1画面に統合します。

表示例:

```text
第3世代 クリア！

ホウエンちほうの ポケモンを
ぜんぶ ゲットしたよ！

第4世代（シンオウ）が
あたらしく 解放されたよ！

[同じレベルでつづける]
[ずかんをみる]
[メニューへもどる]
```

ボタンの意味:

```text
同じレベルでつづける:
- 解放された次世代で、直前と同じレベルのクイズを開始する。

ずかんをみる:
- 図鑑へ遷移する。
- 可能であれば、解放された次世代タブを表示する。
- 第3世代クリア後なら、第4世代タブを開く。

メニューへもどる:
- メニュー画面へ戻る。
```

世代クリア直後には、通常の世代開始画面を挟みません。

理由:

```text
「第◯世代をクリアして、第◯+1世代が解放された」という流れを1画面で伝えた方が分かりやすいため。
```

---

## 18. 全世代コンプリート

第9世代をクリアした場合、全世代コンプリート画面を表示します。

表示例:

```text
全世代コンプリート！

すべての ポケモンを
ゲットしたよ！

[同じレベルでつづける]
[ずかんをみる]
[メニューへもどる]
```

全世代コンプリート後もクイズは継続可能です。

ただし、以下を守ります。

```text
- 新規ポケモン取得は発生しない。
- Firestore 書き込みも発生しない。
- 図鑑は全世代を全開放する。
```

全世代コンプリート後に4問以上正解した場合の表示例:

```text
5もん中 4もん せいかい！

もう ぜんぶ ゲットしているよ！
さんすうは つづけられるよ！

[同じレベルでつづける]
[ずかんをみる]
[メニューへもどる]
```

---

## 19. 図鑑画面

図鑑画面では、以下の世代だけを表示します。

```text
- クリア済み世代
- 現在世代
```

未到達世代は表示しません。

表示例:

```text
[第1世代（カントー）] [第2世代（ジョウト）]
```

取得済みポケモンは、画像と名前を表示します。

未取得ポケモンは、以下のように隠します。

```text
?
???
```

未取得ポケモンをタップした場合は、以下を表示します。

```text
まだ つかまえていないよ
```

---

## 20. 図鑑グリッドのスクロール仕様

図鑑画面全体を長いスクロール画面にしません。

図鑑グリッド部分だけをスクロール領域にします。

```text
- 図鑑ヘッダー
- 世代タブ
- 取得数サマリー
- 図鑑グリッド
- メニューへもどる
```

このうち、スクロールするのは図鑑グリッドだけです。

`メニューへもどる` ボタンは、図鑑グリッドの外に置きます。

`.zukan-grid` の高さ指定は以下です。

```css
max-height: min(58vh, 520px);
max-height: min(58dvh, 520px);
```

---

## 21. ポケモン詳細モーダル

取得済みポケモンをタップすると、詳細モーダルを表示します。

表示項目:

```text
- ポケモン名
- 画像
- タイプ
- 高さ
- 重さ
- 説明
```

モーダル仕様:

```text
- モーダルは .app の外、body直下相当の外側レイヤーに置く。
- モーダル背景は position: fixed。
- 背景クリックで閉じる。
- モーダル本体クリックでは閉じない。
- 右上の × ボタンで閉じる。
- 下部の「とじる」ボタンは表示しない。
- body.modal-open は使わない。
```

重要:

```text
PR #17 で入れた body.modal-open は、実機確認でスクロール挙動を悪化させた。
今後、安易に body.modal-open を戻さないこと。
```

---

## 22. データ読み込み中の表示

ユーザーデータやマスターデータ読み込み中は、明確に読み込み中であることを表示します。

表示例:

```text
よみこみ中...
データを よみこんでいます。
```

図鑑グリッドのマスターデータ読み込み中は、図鑑グリッド内に以下を表示します。

```text
ポケモンデータを よみこんでいます...
```

---

## 23. エラー表示

ユーザーデータやマスターデータが読み込めない場合、エラーを表示します。

表示例:

```text
データを よみこめませんでした

ユーザーデータがまだありません。
```

ユーザーデータは自動作成しません。

```text
データは じどうでは つくりません。
```

## 26. 禁止事項

```text
- 「第◯せだい」表記に戻さない。
- 通常結果画面の「同じレベルでつづける」を「もういちど」に戻さない。
- 保存失敗時の「もういちどゲットする」は消さない。
- 結果画面から「ずかんをみる」を消さない。
- 世代クリア画面と次世代スタート画面を再び分離しない。
- 図鑑画面全体を長いスクロール画面にしない。
- 図鑑モーダルに下部の「とじる」ボタンを戻さない。
- body.modal-open を復活させない。
- migration.html を通常導線に混ぜない。
```

## 27. Main昇格前の確認事項

Main昇格前に、以下を確認します。

```text
- canary/index.html のUXがこの文書と一致している。
- root index.html はまだ V1.5 のままである。
- canary/migration.html は通常導線に混ざっていない。
- users_v2 が V2 の正本として利用されている。
- users は V1.5 の移行元としてのみ扱う。
- 図鑑・モーダルのV1.5回帰修正が維持されている。
- 世代クリア・次世代解放の統合画面が維持されている。
```
