# AGENTS.md

## 1. 基本方針

- このリポジトリは、子ども向けのポケモン算数学習アプリです。
- 本番稼働中の V1.5 は、リポジトリルートの `index.html` です。
- V2 開発は、原則として `canary/` 配下で行います。
- 既存アプリと既存データを壊さないことを最優先にします。
- 実装前に、関連する `docs/` 配下の仕様書・棚卸し文書を確認してください。

## 2. 変更禁止ルール

- 明示的な Issue と承認がない限り、ルートの `index.html` を変更しないでください。
- V2 開発中は、Firestore の `users` コレクションへの書き込みコードを追加・変更しないでください。
- V1 データは、V1 から V2 への移行時の読み取り以外で参照しないでください。
- Firebase 設定を不用意に変更しないでください。
- 既存の `owned` データ構造を破壊しないでください。

## 3. V2 開発ルール

- V2 ユーザーデータは `users_v2` のみを使用します。
- マスターデータは `masters/gen_{1..9}` を読み取り専用として扱います。
- マスターデータはすでに取得済み・Firestore投入済みであることを前提にします。
- V2 本実装では、PokeAPI への直接呼び出しを原則追加しないでください。追加する場合は、理由を PR に明記してください。
- `canary/` 配下にある既存HTMLファイルは、すべて参考用・検討用・作業履歴として扱います。
- V2 本実装・開発用HTMLは、既存の `canary/index.html`、`canary/index_ui.html`、`canary/index_dev_masterdata_2.html` を正本化せず、新規ファイルとして作成してください。
- 新規のV2開発用ファイル名と役割は、実装前に Issue または `docs/` 配下の仕様書で明確にしてください。

## 4. ドキュメントファースト

- 仕様未確定の機能は、先に `docs/` 配下の仕様書を更新してください。
- 1 Issue = 1 Pull Request を原則とします。
- 実装 PR には、対応する仕様書または Issue を必ず紐づけてください。
- 仕様変更を伴う PR では、コードとドキュメントを同時に更新してください。
- 現状認識やファイルの位置づけが変わった場合は、`docs/current_inventory.md` を更新してください。

## 5. UI / UX ルール

- 子どもが読む文言は、原則としてひらがな・カタカナ中心にしてください。
- 漢字を使う場合は、対象年齢に応じて理由を明記してください。
- 保存中・保存完了・保存失敗をわかりやすく表示してください。
- 連打や誤タップを前提に、重要操作には二重実行防止を入れてください。

## 6. PR チェックルール

- `git diff` で変更対象を確認してください。
- 本番ファイル変更の有無を PR 本文に明記してください。
- Firestore の書き込み先を PR 本文に明記してください。
- `/canary` での確認手順を PR 本文に書いてください。
- 画面変更がある場合はスクリーンショットを添付してください。
