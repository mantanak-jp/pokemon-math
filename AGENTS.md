# AGENTS.md

## 1. 基本方針

- このリポジトリは、子ども向けのポケモン算数学習アプリです。
- 現在の本番アプリは、リポジトリルートの `index.html` です。
- `index.html` は V2 Main 本番版です。
- V1.5 本番版は `archive/index_v1_5.html` に退避済みです。
- `canary/` 配下は、V2 Canary 開発・検証の履歴、および今後の検証用として扱います。
- 既存アプリと既存データを壊さないことを最優先にします。
- 実装前に、関連する `docs/` 配下の仕様書・棚卸し文書を確認してください。

## 2. 変更禁止・注意ルール

- 明示的な Issue と承認がない限り、ルートの `index.html` を不用意に変更しないでください。
- `index.html` を変更する場合は、本番 V2 Main への影響を PR 本文に明記してください。
- V2 Main から Firestore の `users` コレクションへ書き込みコードを追加・変更しないでください。
- Firebase 設定を不用意に変更しないでください。
- 既存の `users_v2` データ構造を破壊しないでください。
- 通常ユーザー・子ども向け導線に `canary/migration.html` へのリンクを置かないでください。

## 3. V2 Main / Canary 開発ルール

- V2 ユーザーデータは `users_v2` を正本として使用します。
- マスターデータは `masters/gen_{1..9}` を読み取り専用として扱います。
- V2 本体では、PokeAPI への直接呼び出しを原則追加しないでください。追加する場合は、理由を PR に明記してください。
- `canary/index.html` は、V2 Canary の検証履歴および今後の検証用ファイルです。
- `canary/migration.html` は、運用者向けの V1.5→V2 移行ツールとして残します。
- V1.5 の仕様確認が必要な場合は、`archive/index_v1_5.html` と `docs/system_definition_v1.5.md` を参照してください。

## 4. ドキュメントファースト

- 仕様未確定の機能は、先に `docs/` 配下の仕様書を更新してください。
- 1 Issue = 1 Pull Request を原則とします。
- 仕様変更を伴う PR では、コードとドキュメントを同時に更新してください。
- 現状認識やファイルの位置づけが変わった場合は、`docs/current_inventory.md` を更新してください。

## 5. UI / UX ルール

- 子どもが読む文言は、原則としてひらがな・カタカナ中心にしてください。
- 世代表記は現行仕様どおり「第◯世代」を使用します。
- 保存中・保存完了・保存失敗を分かりやすく表示してください。
- 図鑑詳細モーダルでは `body.modal-open` を安易に戻さないでください。

## 6. PR チェックルール

- `git diff` または GitHub の Files changed で変更対象を確認してください。
- 本番ファイル `index.html` 変更の有無を PR 本文に明記してください。
- Firestore の読み書き先を PR 本文に明記してください。
- `users` / `users_v2` / `masters` の扱いが仕様と合っているか確認してください。
- `canary/migration.html` への通常導線が追加されていないことを確認してください。
